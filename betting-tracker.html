Features

Everything you need to track and analyze sports betting performance.

Dashboard

Live overview: total bets, accumulators, profit, win rate, and ROI. Recent bets listed with results at a glance.

Single Bets & Accumulators

Log bets with match, odds, stake, and type (1X2, Peste/Sub, BTTS). Build multi-match accumulators with automatic total odds and potential profit preview.

Bet History

Full history with Win/Lose/All and Single/Accumulator filters. Update match results individually inside accumulators.

Bet Type Analysis

Performance breakdown per type — 1X2, Peste/Sub, BTTS — with win rate bar, profit, ROI, and "most profitable" highlight.

Profit Chart

Matplotlib chart showing bankroll evolution over time with green/red fill zones for profit and loss periods.

Live Match Data

Upcoming matches from 8 major leagues via football-data.org API. Add matches directly to your bet slip from the UI.

Match Predictor

Predicts win probabilities using last 5 matches form, head-to-head history, standings position, and 1.15x home advantage multiplier.

Value Bet Calculator

Input your estimated probability and bookmaker odds. Get value %, expected value in RON, and a clear "worth it / skip" verdict.

Bank Management

Set initial budget, track current balance, and see total profit/loss. Add funds at any time.

Weekly Summary

Auto-generated weekly report with best/worst bet, total staked, profit, and win rate for the previous week.

Architecture

Clean layered architecture — each layer has one responsibility and communicates only with the layer below.

Layer 1 — UI

GUI + Console

Tkinter dark-themed GUI with sidebar navigation. Also includes a full console interface. Zero business logic in this layer.

Layer 2 — Business

Logic & Analysis

10 modules: tracker, accumulator tracker, value calculator, match predictor, stats analyzer, type analyzer, bank manager, chart, weekly summary.

Layer 3 — Repository

Data Access

File repos for JSON persistence (bets, accumulators, bank), API repo for football-data.org. All I/O is isolated here.

Layer 4 — Domain

Domain Objects

Bet and Accumulator classes with computed profit properties — never stored, always calculated.

Code Highlights

A few interesting implementation details.

Match predictor — scoring algorithm using form, standings & head-to-head:

def calculate_score(form, is_home):
    score = 0
    for match in form:
        # 3 pts for win, 1 for draw, 0 for loss
        match_score = 3 if match["result"] == "W" else 1 if match["result"] == "D" else 0
        match_score += match["goals_scored"] * 0.5 # goal bonus
        match_score -= match["goals_conceded"] * 0.3 # conceded penalty
        score += match_score
    if is_home:
        score *= 1.15 # home advantage multiplier
    return max(score, 0.1)

Value Bet Calculator — expected value formula:

def expected_value(stake, your_probability, bookmaker_odds):
    win_chance = your_probability / 100
    lose_chance = 1 - win_chance
    profit_if_win = stake * bookmaker_odds - stake
    # EV = (win_chance × profit) - (lose_chance × stake)
    ev = (win_chance * profit_if_win) - (lose_chance * stake)
    return round(ev, 2)
